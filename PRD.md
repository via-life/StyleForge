# 风格炼像室 PRD

## 1. 产品目标

- 做一个纯前端、单页、可静态部署的像素头像生成器。
- 用户在同一页完成上传、预览、调参和下载。
- 所有图像处理只在浏览器本地执行，不上传服务器。
- 导出结果与页面主预览严格一致。

## 2. 目标用户

- 想快速生成 GitHub、社交平台或个人主页头像的个人用户。
- 想演示浏览器本地图像处理效果的开发者和设计同学。

## 3. 使用场景

1. 打开网页。
2. 查看顶部 6 个固定风格预览。
3. 上传或拖拽一张本地图片。
4. 选择固定风格或启用自定义风格。
5. 调整背景色、像素颗粒度和主体透明度。
6. 下载最终 `512x512 PNG`。

## 4. 输入与输出

### 输入

- 图片格式：`JPG`、`JPEG`、`PNG`
- 用户交互参数：
  - 固定风格
  - 自定义风格开关与 1 到 3 色映射
  - 背景色
  - 像素颗粒度
  - 主体透明度

### 输出

- 一张固定尺寸的 `512x512 PNG`
- 导出内容必须与中间 `GlassSurface` 主预览一致

## 5. 页面结构

页面保持单页纵向结构，无路由、无模态：

1. 顶部品牌区
2. 上方 `CircularGallery` 固定风格预览区
3. 中间 `GlassSurface` 上传 / 主预览区
4. 主预览下方直属调节区
5. 底部输出动作区
6. `Toast` 提示区

## 6. 当前核心功能

### 6.1 上传

- 支持点击上传
- 支持整页拖拽上传
- 只允许单张图片
- 非法格式和多文件上传时显示错误提示

### 6.2 固定风格系统

固定 6 个风格：

- 普通
- 清新
- 赛博
- 多巴胺
- 暗黑
- 玻璃

要求：

- 顶部始终只显示这 6 个固定风格
- 点击风格卡切换主预览
- 风格带内部使用重复数组实现顺滑循环
- 桌面端优先露出 5 张，小屏根据宽度减少同时可见数
- 玻璃风格只作用于主体，不改变背景层

### 6.3 自定义风格模块

- 自定义风格不进入顶部风格带
- 位于中间主预览正下方
- 默认关闭
- 开启后覆盖当前固定风格的主预览与导出
- 顶部 6 张固定风格卡不跟随自定义配色变化
- 支持颜色数量：`1 / 2 / 3`
- 每个颜色支持取色器和 `RGB` 输入

映射规则：

- 1 色：单色调映射
- 2 色：阴影映射颜色 1，高光映射颜色 2
- 3 色：阴影、中间调、高光分别映射颜色 1、2、3

### 6.4 背景色

- 默认背景为透明
- 提供多个预设背景色
- 提供自定义背景色
- 自定义背景色支持取色器和 `RGB` 输入
- 背景色只改变背景层，不改变主体风格颜色
- 顶部固定风格预览和中间主预览同步响应背景切换

### 6.5 像素颗粒度控制

- 不保留 `Crop + Zoom`
- 使用离散滑杆控制像素颗粒度
- 档位固定为：
  - `512`
  - `256`
  - `128`
  - `64`
  - `32`
  - `16`
  - `8`
  - `5`
- 默认值为 `512`
- 越往右像素越粗
- 顶部固定风格预览和主预览同步跟随颗粒度变化

### 6.6 主体透明度控制

- 使用 `0% - 100%` 滑杆控制主体透明度
- 透明度只作用于主体，不作用于背景层
- 顶部固定风格缩略图和主预览同步跟随透明度变化

### 6.7 主预览与导出

- 主预览固定按 `512x512` 结果渲染
- 主预览外层保留识别边框，但边框不进入导出图
- 导出格式固定为 `PNG`
- 下载图片与当前 `GlassSurface` 主预览完全一致

## 7. 当前图像处理流程

固定顺序：

1. 读取本地图片
2. 完整显示图片，四周留空或留背景色，不做裁切缩放
3. 提取图片主体，背景初始为透明
4. 对主体执行固定风格或自定义风格渲染
5. 按当前颗粒度做像素化
6. 应用主体透明度
7. 再填充背景色
8. 输出顶部固定风格预览、中间主预览和导出图

## 8. 非功能要求

- 技术栈：`React + Vite + OGL`
- 部署方式：静态部署，兼容 GitHub Pages
- 隐私要求：图片不上传服务器
- 体验要求：
  - 所有操作在同一页完成
  - 风格切换、背景切换、颗粒度切换、透明度切换后主预览应及时刷新
- 性能要求：
  - 桌面端保留完整视觉效果
  - 移动端允许降级背景动画、风格带和玻璃特效，但主流程功能必须保留

## 9. 成功标准

- 用户能在同一页内完成上传、切换固定风格、启用自定义风格、切换背景、调整颗粒度、调整主体透明度和下载
- 顶部 6 张固定风格卡都能正常生成并循环展示
- 自定义风格只影响中间主预览和导出，不污染顶部固定风格带
- 背景色切换只改变背景，不污染主体颜色
- 导出结果与中间主预览一致
- 断网后仍可完成本地主要流程
