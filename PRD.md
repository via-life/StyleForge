# 风格炼像室 PRD

## 1. 产品目标

- 做一个纯前端、单页、可静态部署的像素头像生成器。
- 用户在同一页内完成上传、预览、调参和下载。
- 所有图像处理只在浏览器本地执行，不上传服务器。

## 2. 目标用户

- 想快速生成 GitHub / 社交头像的个人用户。
- 想演示前端图像处理效果的开发者或设计同学。

## 3. 使用场景

1. 打开网页。
2. 查看默认 5 种风格预览。
3. 上传或拖拽一张本地图片。
4. 选择风格、背景色、像素颗粒度。
5. 下载最终 `512x512 PNG`。

## 4. 输入与输出

### 输入

- 图片格式：`JPG`、`JPEG`、`PNG`
- 用户交互参数：
  - 风格
  - 背景色
  - 像素颗粒度

### 输出

- 一张固定尺寸的 `512x512 PNG`
- 导出内容必须与中间 `GlassSurface` 主预览一致

## 5. 页面结构

页面保持单页纵向结构，无路由、无模态：

1. 顶部品牌区
2. 上方 `CircularGallery` 风格预览区
3. 中间 `GlassSurface` 上传 / 主预览区
4. 底部控制区
5. `Toast` 提示区

## 6. 当前核心功能

### 6.1 上传

- 支持点击上传
- 支持整页拖拽上传
- 只允许单张图片
- 非法格式和多文件上传时显示错误提示

### 6.2 风格系统

固定 5 个风格：

- 普通
- 清新
- 赛博
- 多巴胺
- 暗黑

要求：

- 上传后顶部始终保留这 5 个风格
- 点击风格卡或下方标签切换主预览
- 风格卡内部使用重复数组实现顺滑无限循环
- 首屏尽量优先显示 5 个唯一风格
- 小屏根据宽度减少同时可见张数，但保留顺滑循环

### 6.3 背景色

- 默认背景为透明
- 提供多个预设背景色
- 背景色只改变背景层，不改变主体风格颜色
- 顶部风格预览和中间主预览同步响应背景切换

### 6.4 像素颗粒度控制

- 不保留 `Crop + Zoom`
- 使用离散滑杆控制像素颗粒度
- 档位固定为：
  - `512`
  - `256`
  - `128`
  - `64`
  - `32`
  - `16`
  - `8`
  - `5`
- 默认值为 `512`
- 越往右像素越粗
- 顶部风格预览和主预览同步跟随颗粒度变化

### 6.5 主预览与导出

- 主预览固定按 `512x512` 结果渲染
- 导出格式固定为 `PNG`
- 下载图片与当前 `GlassSurface` 主预览完全一致

## 7. 当前图像处理流程

固定顺序：

1. 读取本地图片
2. 完整显示图片，四周留空或留背景色，不做裁切缩放
3. 提取图片主体，背景初始为透明
4. 对主体执行风格化
5. 按当前颗粒度做像素化
6. 再填充背景色
7. 输出顶部风格预览、中间主预览和导出图

## 8. 非功能要求

- 技术栈：`React + Vite + OGL`
- 部署方式：静态部署，兼容 GitHub Pages
- 隐私要求：图片不上传服务器
- 体验要求：
  - 所有操作在同一页完成
  - 风格切换、背景切换、颗粒度切换后主预览应及时刷新

## 9. 成功标准

- 用户能在同一页内完成上传、切换风格、切换背景、调整颗粒度和下载
- 上传后的 5 张风格卡都能正常生成
- 背景色切换只改变背景，不污染主体颜色
- 导出结果与中间主预览一致
- 断网后仍可完成本地主要流程
